# HH-бот — парсер вакансий hh.ru с авто-откликом (веб-интерфейс)

Локальный сайт, который ищет вакансии на hh.ru по вашим критериям и **сам откликается**
на подходящие. Работает не через API, а управляя настоящим браузером (Playwright):
вы один раз логинитесь вручную, дальше бот кликает за вас. Интерфейс — тёмная
админ-панель в браузере.

## Установка

```bash
pip install -r requirements.txt
playwright install chromium
```

## Запуск

```bash
python main.py
```

Откроется вкладка `http://127.0.0.1:8000`. Браузер, которым управляет бот, по
умолчанию работает **невидимо** (в фоне) и не мешает на экране.

### Видимое окно браузера

При нажатии **«Войти»** бот сам открывает видимое окно браузера, чтобы вы могли
залогиниться на hh.ru вручную. После входа сессия сохраняется, и дальше всё
работает в фоне.

Если нужно, чтобы окно браузера было видимым **всегда** (например для отладки),
запустите с переменной окружения `HH_HEADLESS=0`:

```bash
# Windows (PowerShell)
$env:HH_HEADLESS = "0"; python main.py
```

## Как пользоваться

1. В шапке справа — статус входа. Нажмите **«Войти»** и залогиньтесь на hh.ru вручную
   в открывшемся окне (логин/пароль, при необходимости капча и SMS). Сессия сохранится.
2. Заполните **критерии** (профессии с живыми подсказками, город, зарплата, исключающие
   слова, письмо, лимит) — **«Сохранить»** запоминает их в `config.yaml`.
3. **«🔍 Найти»** — собирает подходящие вакансии в таблицу (строгий отбор по профессии).
4. **«🚀 Запустить отклики»** — бот откликается сам, с паузами и в пределах лимита.
   **«⏹ Стоп»** — прервать.
5. **«Мои отклики»** — история того, на что вы уже откликались.
6. **«Ответы на отклики»** — что ответили работодатели.

## Настройки (`config.yaml`)

| Поле | Значение |
|------|----------|
| `professions` | список поисковых запросов (профессий) |
| `region` | id региона hh.ru (1 = Москва, 2 = СПб, 113 = Россия) |
| `salary_min` | минимальная зарплата, 0 = не фильтровать |
| `exclude_words` / `include_words` | слова-стопы и обязательные слова в названии |
| `cover_letter` | текст сопроводительного письма |
| `daily_limit` | дневной лимит откликов (лимит hh.ru ~200) |
| `delay_seconds` | пауза между откликами `[мин, макс]` |
| `max_pages` | сколько страниц выдачи обходить на запрос |
| `strict_title_match` | строгий отбор: профессия должна быть в названии вакансии |

## Важно

- **Лимиты и блокировки.** hh.ru ограничивает число откликов в день (~200) и может
  блокировать за подозрительную активность. Не снижайте паузы и держите лимит ниже 200.
- **Капча.** Если при отклике появляется капча, вакансия помечается «ошибка» —
  пройдите капчу вручную в окне браузера и запустите снова.
- **Хрупкость.** Вёрстка hh.ru меняется. Если перестало находить элементы — правьте
  селекторы в `hh_bot/selectors.py`.
- Память откликов хранится в `history.db` — на одну вакансию дважды не откликнётесь.
- Используйте ответственно и в рамках правил сервиса hh.ru.

## Структура

```
main.py                 точка входа (запуск сайта)
config.yaml             ваши критерии
hh_bot/
  config.py             загрузка/сохранение критериев
  models.py             модель вакансии
  browser.py            Playwright: сессия и логин
  selectors.py          все селекторы hh.ru
  search.py             построение URL и парсинг выдачи
  filters.py            отбор по критериям (синонимы профессий)
  applier.py            авто-отклик
  responses.py          сбор ответов работодателей
  storage.py            SQLite-история (дедупликация)
  suggest.py            живые подсказки профессий hh.ru
  worker.py             рабочий поток (владеет браузером)
web/
  server.py             Flask: API + поток событий (SSE)
  static/               index.html, style.css, app.js — тёмная админ-панель
```
